# 📋 Material for MkDocs 提示框完整指南

Material for MkDocs 提供了丰富的提示框(Admonitions)样式，可以让您的文档更加美观和易读。本文档展示所有可用的提示框类型和用法。


<https://squidfunk.github.io/mkdocs-material/reference/admonitions/>
---

## 🔥 基础提示框类型

### Note - 说明提示

!!! note
    这是一个基础的说明提示框，用于提供额外的信息或说明。

!!! note "自定义标题"
    您可以为提示框添加自定义标题，只需在类型后面加上引号和标题文本。

### Info - 信息提示

!!! info
    信息提示框用于展示重要的信息内容，通常是中性的说明。

!!! info "重要信息"
    这种提示框适合放置产品信息、版本说明或一般性的重要内容。

### Tip - 技巧提示

!!! tip
    技巧提示框用于分享有用的小贴士、最佳实践或专业建议。

!!! tip "专业提示"
    使用这种提示框来突出显示能够帮助用户更好使用产品的技巧。

### Success - 成功提示

!!! success
    成功提示框用于表示某个操作已成功完成或展示正面的结果。

!!! success "操作完成"
    当用户完成某个重要步骤时，可以使用这种样式来给予积极反馈。

### Warning - 警告提示

!!! warning
    警告提示框用于提醒用户注意潜在的问题或需要谨慎操作的内容。

!!! warning "重要警告"
    在用户可能犯错误或需要特别注意的地方使用这种提示框。

### Failure - 失败/错误提示

!!! failure
    失败提示框用于展示错误信息、失败的操作或需要避免的情况。

!!! failure "操作失败"
    当某个操作失败或用户需要了解错误信息时使用。

### Danger - 危险提示

!!! danger
    危险提示框用于警告用户可能造成严重后果的操作或情况。

!!! danger "严重警告"
    这是最高级别的警告，用于可能导致数据丢失或系统损坏的情况。

### Bug - 错误提示

!!! bug
    错误提示框用于报告已知的bug、缺陷或需要修复的问题。

!!! bug "已知问题"
    用于记录软件中的已知问题或限制。

## 🎨 扩展提示框类型

### Example - 示例提示

!!! example
    示例提示框用于展示代码示例、使用案例或具体的操作演示。

!!! example "代码示例"
    ```python
    def hello_world():
        print("Hello, World!")
    ```

### Quote - 引用提示

!!! quote
    引用提示框用于展示引文、名言或来自其他来源的内容。

!!! quote "名人名言"
    "代码是写给人看的，只是顺便能在机器上运行。" - Martin Fowler

### Abstract - 摘要提示

!!! abstract
    摘要提示框用于展示文档摘要、总结或概述内容。

!!! abstract "章节摘要"
    本章节介绍了Material for MkDocs中所有可用的提示框类
